1. ANYON이란

ANYON은 무엇인가요?

ANYON은 AI 파워드 개발 플랫폼입니다. 코딩 지식이 없어도 누구나 소프트웨어를 만들 수 있도록 도와주는 데스크톱 애플리케이션입니다.

기존에는 좋은 아이디어가 있어도 개발자를 찾거나, 직접 배워야 했습니다. ANYON은 Claude 4.5 AI를 파트너로 삼아, 아이디어부터 실제 코드까지 전체 개발 과정을 자동화합니다.

마치 경험 많은 개발자와 함께 일하는 것처럼, AI가 기획 문서를 작성하고, 시스템을 설계하고, 코드를 생성해줍니다.

핵심 메시지: "아이디어에서 프로덕션까지. 코드 없이."
기획부터 배포까지 모든 과정을 AI와 함께 완성할 수 있습니다.

핵심 가치

1. 완전한 개발 워크플로우

ANYON은 단순한 코드 생성 도구가 아닙니다. 전문 개발팀의 전체 프로세스를 자동화합니다:

  • 기획 단계: 제품 요구사항(PRD) 작성
  • 설계 단계: UX 기획, UI 디자인, 기술 검토, 아키텍처 설계, 데이터 모델
  • 개발 단계: 백엔드, 프론트엔드 코드 자동 생성
  • 배포 단계: 코드 리뷰 및 최적화

2. AI 파트너와의 협업

Claude 4.5는 단순히 명령을 따르는 도구가 아닙니다. 대화를 통해 여러분의 아이디어를 함께 다듬고, 개선하고, 구현합니다.

"펫시터 앱을 만들고 싶어"라고 말하면, AI가 "어떤 기능이 필요할까?", "사용자는 누구야?", "어떻게 돈을 버는가?" 등의 질문을 통해 기획을 함께 만들어갑니다.

3. 개발 시간 90% 단축

일반적으로 중소 프로젝트는:

  • 개발자 모집: 2주
  • 기획 및 설계: 2주
  • 개발: 4-8주
  • 테스트 및 배포: 1주
  • 총 9-13주 (약 3개월)

ANYON으로는:

  • 설치: 5분
  • 아이디어 설명: 1시간
  • 기획 워크플로우: 2시간
  • 코드 생성 및 배포: 1시간
  • 총 4시간

4. 로컬에서 안전하게 작업

ANYON은 데스크톱 애플리케이션입니다. 여러분의 모든 프로젝트와 데이터가 로컬 컴퓨터에 저장되므로 완전한 통제가 가능합니다. 클라우드 의존성이 없으므로 개인 정보 보호도 좋습니다.

5. 오픈소스 활용

"비슷한 앱이 있을까?"라고 생각하신다면, ANYON이 GitHub에서 찾아줍니다. 기존 오픈소스 프로젝트를 기반으로 시작하면 개발 시간을 더욱 줄일 수 있습니다.

타겟 사용자

누가 ANYON을 사용할까요?

1️⃣ 비즈니스 아이디어가 있는 창업자 / 기획자

"좋은 아이디어가 있는데 개발자를 구할 시간과 돈이 없다"
→ ANYON으로 직접 MVP를 만들어 시장 검증을 할 수 있습니다.

2️⃣ 빠른 프로토타이핑이 필요한 팀

"새로운 기능을 빠르게 만들어서 테스트해봐야 해"
→ ANYON으로 프로토타입을 며칠 안에 완성할 수 있습니다.

3️⃣ 개발 생산성을 높이려는 개발자

"보일러플레이트 코드 작성은 반복적이고 지루해"
→ ANYON으로 AI가 기초 코드를 생성하고, 개발자는 핵심 로직에만 집중할 수 있습니다.

4️⃣ 기술을 배우고 싶은 초보자

"소프트웨어 개발이 어떻게 진행되는지 배우고 싶어"
→ ANYON으로 실제 개발 프로세스(기획→설계→개발)를 경험할 수 있습니다.

2. ANYON의 기능

6단계 기획 워크플로우

ANYON은 전문 제품팀이 하는 체계적인 기획 과정을 자동화합니다. 아이디어를 말씀해주면, AI가 각 단계에서 필요한 문서를 작성해줍니다.

Step 1: PRD (Product Requirements Document)

제품 요구사항을 정의하는 문서입니다. "누가 쓸 거야?", "왜 필요해?", "어떤 기능을 포함할래?" 등을 정리합니다.

AI와의 대화를 통해 "펫시터 앱"이라는 간단한 아이디어가 "펫 소유자와 펫시터를 연결하는 플랫폼으로, 예약, 결제, 리뷰 기능을 포함한다"라는 명확한 제품정의로 확장됩니다.

Step 2: UX (User Experience)

사용자가 어떻게 앱을 사용할지 기획합니다. 어떤 화면이 필요하고, 어떻게 흐르는지를 정리합니다.

"로그인 → 펫시터 검색 → 예약 → 결제 → 리뷰" 같은 사용자 여정을 만듭니다.

Step 3: UI (User Interface)

화면 디자인입니다. 각 화면에서 버튼, 텍스트, 이미지가 어디에 배치될지 정합니다.

ANYON은 이 문서를 바탕으로 React 컴포넌트를 생성할 수 있습니다.

Step 4: TRD (Technical Requirements Document)

기술적 결정사항을 정합니다. "데이터베이스는 뭘 쓸래?", "API는 어떻게 설계할래?" 같은 것들입니다.

AI가 "이 기능에는 PostgreSQL이 좋고, REST API로 설계하면 된다"는 식으로 조언해줍니다.

Step 5: Architecture (시스템 설계)

전체 시스템의 구조를 정합니다. 서버, 데이터베이스, 클라이언트가 어떻게 연결되는지를 그립니다.

Step 6: ERD (Entity Relationship Diagram)

데이터베이스 테이블과 관계를 정합니다. "User 테이블과 Pet 테이블이 어떻게 연결되는가?"를 정합니다.

ANYON은 이 ERD를 바탕으로 Prisma 스키마를 자동 생성합니다.

6단계의 장점:
• 생각을 명확하게 정리할 수 있습니다
• AI가 이해하기 쉬워서 정확한 코드 생성이 가능합니다
• 나중에 변경하기 쉽습니다 (문서만 수정하면 코드 재생성)

4단계 개발 자동화

기획이 완료되면, ANYON이 실제 코드를 자동으로 생성합니다. 4단계로 진행됩니다:

Step 1: PM Opensource

ANYON이 GitHub에서 비슷한 오픈소스 프로젝트를 찾아줍니다. "펫시터 앱 같은 마켓플레이스 플랫폼 있을까?"라고 검색합니다.

기존 프로젝트를 기반으로 시작하면 처음부터 만드는 것보다 훨씬 빠릅니다.

Step 2: PM Orchestrator

기획 문서를 실행 가능한 개발 티켓으로 변환합니다.

예: "사용자가 펫시터를 검색할 수 있어야 한다" → "검색 API 개발", "검색 UI 컴포넌트 만들기", "검색 테스트 작성" 등 여러 티켓으로 분해합니다.

Step 3: PM Executor

각 티켓을 실행하여 코드를 작성합니다. 백엔드 API, 프론트엔드 컴포넌트, 테스트 코드까지 모두 생성합니다.

이때 AI가 작성하는 코드는:

  • 타입이 안전합니다 (TypeScript)
  • 테스트 가능합니다 (테스트 코드 포함)
  • 프로덕션 준비가 됩니다 (에러 처리, 로깅 포함)

Step 4: PM Reviewer

생성된 코드를 자동으로 리뷰합니다. "이 코드는 성능이 좋을까?", "보안 문제는 없을까?", "더 간단한 방법은 없을까?" 등을 확인합니다.

개선 사항이 있으면 자동으로 코드를 개선합니다.

예시: 펫시터 앱 개발

기획 문서: "사용자는 펫시터를 검색하고 예약할 수 있어야 한다"

생성되는 코드:
• 백엔드: GET /sitters (검색 API), POST /bookings (예약 API)
• 프론트엔드: SearchSitter.tsx, BookingForm.tsx
• 데이터베이스: Sitter, Booking 테이블
• 테스트: search.test.ts, booking.test.ts

소요 시간: 1-2시간 (수동 개발은 며칠 소요)

핵심 기능

1. 에이전트 (Agents)

커스텀 AI 에이전트를 만들 수 있습니다. 반복되는 작업을 자동화하기 위해 자신만의 AI 조수를 만드는 것입니다.

예: "매일 오전 10시에 테스트를 돌리고 결과를 이메일로 보내줘" 같은 에이전트를 만들 수 있습니다.

2. MCP (Model Context Protocol)

외부 서비스와 연결할 수 있습니다. GitHub, Slack, Google Drive 등과 연동하여 더 강력한 워크플로우를 만들 수 있습니다.

3. 세션 관리

작업 중단 후 복구 가능합니다. 체크포인트를 저장해두면, 나중에 정확히 같은 지점에서 계속 작업할 수 있습니다.

4. 사용량 추적

실시간으로 Claude API 사용량을 확인할 수 있습니다. "지금까지 몇 개의 토큰을 썼고, 비용이 얼마가 될지" 한눈에 봅니다.

기술 스택

ANYON의 기술 구성은 다음과 같습니다:

레이어 기술 이유
AI 모델 Claude 4.5 최신 AI로 정확하고 신뢰할 수 있는 코드 생성

3. 사용자 가이드

시작하기

1단계: 설치

  1. GitHub Releases에서 OS에 맞는 버전 다운로드
  2. 인스톨러 실행 (macOS .dmg, Windows .exe, Linux .AppImage)
  3. ANYON 실행

2단계: 인증

  1. ANYON을 처음 열면 로그인 화면이 나옵니다
  2. "Google로 로그인" 버튼을 클릭
  3. Google 계정으로 인증
  4. 완료! 이제 사용할 준비가 됐습니다

3단계: Claude 설정

  1. ANYON 설정에서 "Claude API" 메뉴로 이동
  2. Claude API 키 발급 (Anthropic 웹사이트)
  3. API 키를 ANYON에 입력
  4. 이제 Claude와 대화할 수 있습니다!
⚠️ 주의: API 키는 절대로 타인과 공유하지 마세요. 키가 노출되면 비용이 청구될 수 있습니다.

워크플로우 따라하기

예제: 간단한 TODO 앱 만들기

Step 1: 프로젝트 생성

동작: ANYON 메인 화면에서 "새 프로젝트" 클릭

입력: 프로젝트 이름 "SimpleTodo"

결과: 빈 프로젝트 생성 완료

Step 2: 아이디어 설명

동작: Claude와의 대화 시작

입력:

"간단한 TODO 앱을 만들고 싶어.
사용자가 할 일을 추가하고, 삭제하고, 완료 표시할 수 있는 앱이야.
웹으로 접속하는 방식이야."

Claude의 응답: "좋아! 더 자세하게 물어볼께: - 몇 명이 사용할 거야? - 할 일에 마감일을 설정하고 싶어? - 완료된 할 일은 자동으로 삭제할까?"

Step 3: PRD 작성

동작: ANYON의 "기획" 탭에서 "PRD 작성" 클릭

Claude가 자동으로 작성:

# SimpleTodo 제품 요구사항

## 핵심 기능
1. 할 일 추가 (제목, 설명, 마감일)
2. 할 일 조회 (모든 할 일, 진행 중, 완료됨)
3. 할 일 업데이트 (제목, 설명, 마감일 수정)
4. 할 일 삭제
5. 완료 표시

## 사용자
- 학생: 과제 관리
- 직장인: 일일 업무 관리

확인: "이게 맞니?" 하고 물어보면 "네!" 또는 수정

Step 4: UX 기획

동작: "UX" 탭에서 "UX 기획 작성"

Claude가 자동으로 작성:

# SimpleTodo UX 흐름

1. 홈 페이지
   - 오늘의 할 일 3개 표시
   - "새로운 할 일" 버튼
   - 필터: 모든 할 일 / 진행 중 / 완료됨

2. 할 일 추가 페이지
   - 제목 입력
   - 설명 입력
   - 마감일 선택
   - 저장 버튼

3. 할 일 상세 페이지
   - 할 일 내용 표시
   - 완료 표시 버튼
   - 수정 버튼
   - 삭제 버튼

Step 5: UI 디자인

동작: "UI" 탭에서 "UI 디자인 생성"

Claude가 자동으로 작성:

# UI 레이아웃

## 홈 페이지
┌────────────────────────┐
│ SimpleTodo             │  (헤더)
├────────────────────────┤
│ 📝 오늘의 할 일        │
│ ☐ 과제 제출            │  (할 일 카드)
│ ☐ 메일 회신            │
│ ☐ 회의 준비            │
│                        │
│ [+ 새로운 할 일]       │  (버튼)
│ [모든 할 일] [완료] │  (필터)
└────────────────────────┘

Step 6-8: 기술 설계 → 코드 생성

동작: "개발 시작" 버튼 클릭

ANYON이 자동으로:

ANYON에서 자신만의 AI 에이전트를 만들 수 있습니다. 예를 들어:

  • 테스트 에이전트: 매일 자동으로 테스트를 돌리고 결과를 리포트
  • 문서화 에이전트: 코드가 바뀔 때마다 자동으로 문서 업데이트
  • 배포 에이전트: "배포해줘"라고 말하면 자동으로 배포

자신의 워크플로우에 맞게 에이전트를 커스터마이징할 수 있습니다.

자주 묻는 질문 (FAQ)

Q: 정말 코딩 지식이 없어도 되나요?

A: 네! ANYON이 모든 기술적 결정과 코드 작성을 담당합니다. 여러분은 아이디어와 요구사항만 설명하면 됩니다. 물론 기술 지식이 있으면 더 정교한 요청을 할 수 있습니다.

Q: 생성된 코드의 품질은 어떻게 되나요?

A: Claude 4.5는 매우 능력 있는 AI입니다. 생성되는 코드는:

  • 타입 안전함 (TypeScript)
  • 테스트 가능함 (테스트 코드 포함)
  • 프로덕션 준비됨 (에러 처리, 로깅)
  • 성능 최적화됨

물론 100% 완벽할 수는 없으므로, 자신의 코드를 리뷰하고 수정할 수 있습니다.

Q: 생성된 코드는 제 것인가요?

A: 네! 생성된 모든 코드는 여러분의 소유입니다. ANYON은 AGPL-3.0 라이선스 하에 제공되며, 생성된 코드는 여러분이 자유롭게 수정하고 배포할 수 있습니다.

Q: 인터넷이 없으면 사용 불가능한가요?

A: Claude API를 사용하려면 인터넷이 필요합니다. 다만 기획 문서 작성, 코드 검토 등 일부 기능은 오프라인 모드로 진행할 수 있습니다.

Q: 여러 개의 프로젝트를 동시에 관리할 수 있나요?

A: 네! ANYON은 프로젝트 관리 기능을 제공합니다. 여러 프로젝트를 동시에 작업할 수 있고, 각 프로젝트의 진행 상황을 추적할 수 있습니다.

Q: 데이터가 안전한가요?

A: 네! ANYON은 데스크톱 애플리케이션이므로 모든 프로젝트 데이터가 로컬 컴퓨터에 저장됩니다. 클라우드 서버에 저장되지 않으므로 개인 정보가 보호됩니다. 다만 Claude API 호출 로그는 Anthropic 서버에 저장될 수 있습니다.
