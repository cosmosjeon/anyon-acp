## 예시 결과물 (prd.md)

```markdown
# PRD (Product Requirements Document)

**작성일:** 2025-12-19

---

## 1. 제품 개요

**서비스 유형:** 모바일 앱 (iOS/Android)

**한 줄 정의:** 반려동물 보호자와 검증된 펫시터를 연결하는 매칭 플랫폼

---

## 2. 해결하려는 문제

- 출장/여행 시 반려동물을 맡길 곳이 없음
- 지인에게 부탁하기 미안함
- 펫호텔은 비싸고 반려동물이 스트레스 받을까 걱정
- 기존 펫시터 앱은 검증이 부족해서 믿음이 안 감
- 맡기고 나면 우리 아이가 어떻게 지내는지 알 수 없어서 불안

---

## 3. 타겟 사용자

- 출장이 잦은 직장인
- 반려동물을 키우면서 급하게 돌봄이 필요한 상황이 자주 생기는 사람
- 기존 펫시터 앱을 써봤지만 신뢰 문제로 불만족한 사람

---

## 4. 핵심 기능

### MVP 필수 기능
1. **펫시터 검증 시스템** - 신분증, 범죄이력, 교육이수 확인해서 믿을 수 있는 펫시터만
2. **실시간 사진 리포트** - 돌봄 중 사진 보내서 보호자가 안심할 수 있게
3. **날짜/위치 기반 검색** - 원하는 날짜에 가까운 펫시터 찾기
4. **결제/정산 시스템** - 안전하게 결제하고 펫시터에게 정산

### 향후 추가 기능 (v2+)
- 즉시 매칭 - 급할 때 오늘 가능한 펫시터 바로 찾기
- 돌봄 일지 - 밥, 산책 등 기록
- 채팅 기능 - 예약 전 펫시터랑 대화
- 리뷰/평점 - 다른 보호자들 후기 참고

---

## 5. 참고 오픈소스

**Firebase (실시간 리포트 기능)**
- Cloud Messaging으로 실시간 알림
- Cloud Storage로 사진 저장 및 전송
- https://firebase.google.com

---

## 6. 다음 단계

이 PRD를 바탕으로:
1. **UX 디자인** - 화면 구성과 사용자 흐름 설계
2. **기술 설계** - 어떤 기술로 만들지 결정
3. **개발 시작** - 실제 코딩

을 진행합니다.
```

## 플로우 특징

### 대화형 구체화의 장점

| 특징 | 설명 |
|------|------|
| 구체적인 선지 | "직장인" 대신 "출장이 잦아서 급하게 펫시터 구해야 할 때가 많은 직장인" |
| 자유 답변 옵션 | 모든 질문에 "또는 다르게 생각하시는 게 있으면 말씀해주세요" |
| 충분한 질문 | 8-10개 질문으로 아이디어 깊이 있게 탐색 |
| 왜 필요한지 함께 | 기능 나열 시 "~해서 ~하게" 형식으로 이유 포함 |

### 질문 설계 원칙

1. **선지는 구체적으로** - 상황과 이유 포함
2. **항상 자유 답변 옵션** - 주관식으로도 답할 수 있게
3. **질문 자체가 힌트** - 사용자가 미처 생각 못한 관점 제시

### 핵심 원칙

1. **대화를 통한 구체화** - 질문을 통해 사용자가 스스로 생각을 정리하게
2. **선택지 + 자유 답변** - 객관식은 힌트
3. **피드백/평가 X** - 구체화만
4. **비개발자 친화적** - 쉬운 말로
