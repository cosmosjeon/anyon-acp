# UX Design 워크플로우

## 개요

UX Design 워크플로우는 PRD를 바탕으로 사용자 경험(User Experience)을 설계하는 두 번째 단계입니다. 실제 사용자가 제품을 어떻게 사용할지 흐름을 설계합니다.

## 파일 위치
- **프롬프트 파일**: `src/constants/workflows/planning/startup-ux.ts`
- **출력 파일**: `anyon-docs/planning/ux-design.md`
- **참조 파일**: `anyon-docs/planning/prd.md`

---

## 워크플로우 진행 순서

### Step 1: PRD 분석

AI가 수행하는 작업:
- PRD 문서 읽기
- 핵심 기능 파악
- 타겟 사용자 이해

### Step 2: 사용자 플로우 정의

AI가 물어보는 질문들:
- 사용자가 앱에 처음 들어오면 어떤 화면을 보나요?
- 핵심 기능까지 도달하는 경로는?
- 회원가입/로그인 플로우는 어떻게?

**결과물**: 사용자 플로우 다이어그램

### Step 3: 화면 구성 정의

각 화면에 대해:
- 화면 이름
- 화면 목적
- 포함되는 요소들
- 다음 화면으로 이동하는 방법

### Step 4: 네비게이션 구조

정리하는 내용:
- 메인 메뉴 구조
- 탭 구성
- 뒤로가기/홈 이동 로직

### Step 5: 인터랙션 패턴

정의하는 내용:
- 버튼 클릭 시 동작
- 스와이프 제스처
- 로딩/에러 상태 처리

### Step 6: UX Design 문서 생성

모든 정보를 종합하여 `ux-design.md` 파일 생성

---

## AI의 대화 스타일

### 페르소나
- **역할**: UX 디자인 전문가
- **접근법**: 사용자 중심 사고
- **목표**: 직관적이고 사용하기 쉬운 경험 설계

### 질문 방식
```
사용자가 [특정 기능]을 사용할 때:

1. 어떤 화면에서 시작하나요?
2. 몇 번의 클릭/탭이 필요한가요?
3. 완료 후 어디로 이동하나요?
```

## 출력 문서 구조 (ux-design.md)

```markdown
# [프로젝트명] UX Design

## 1. 사용자 플로우
- 메인 플로우 다이어그램
- 서브 플로우 정리

## 2. 화면 목록
- 각 화면 상세 설명
- 화면 간 관계

## 3. 네비게이션 구조
- 메뉴 구조도
- 이동 경로

## 4. 인터랙션 패턴
- 공통 인터랙션
- 화면별 특수 인터랙션

## 5. 상태 처리
- 로딩 상태
- 에러 상태
- 빈 상태
```

## 핵심 개념 설명

### 사용자 플로우 (User Flow)
사용자가 앱에서 특정 목표를 달성하기까지의 경로입니다.
```
시작 → 화면A → 화면B → 목표 달성
```

### 와이어프레임
화면의 대략적인 레이아웃을 그린 것입니다. 색상이나 디자인 없이 구조만 표현합니다.

### 인터랙션
사용자가 화면과 상호작용하는 방식입니다. (탭, 스와이프, 입력 등)
