# PRD (Product Requirements Document) 워크플로우

## 개요

PRD 워크플로우는 사용자의 아이디어를 체계적인 제품 요구사항 문서로 변환하는 첫 번째 단계입니다.

## 파일 위치
- **프롬프트 파일**: `src/constants/workflows/planning/startup-prd.ts`
- **출력 파일**: `anyon-docs/planning/prd.md`

---

## 워크플로우 진행 순서

### Step 1: 프로젝트 기본 정보 수집

AI가 물어보는 질문들:
- 프로젝트 이름이 무엇인가요?
- 어떤 문제를 해결하려고 하나요?
- 목표 사용자(타겟)는 누구인가요?

**특징**: 4~8개의 객관식 선택지 제공, `(추천)` 표시로 가이드

### Step 2: 핵심 기능 정의

AI가 물어보는 질문들:
- 이 제품의 핵심 기능은 무엇인가요?
- MVP(최소 기능 제품)에 꼭 필요한 기능은?
- 나중에 추가할 기능은?

### Step 3: 사용자 스토리 작성

각 핵심 기능에 대해:
- "사용자로서 나는 [기능]을 원한다. 왜냐하면 [이유]이기 때문이다."
- 형식으로 사용자 스토리 정리

### Step 4: 경쟁사 분석 (WebSearch 사용)

**중요**: 이 단계에서 AI가 직접 웹 검색을 수행합니다.

검색 내용:
- 비슷한 서비스/제품 조사
- 경쟁사의 강점과 약점 분석
- 차별화 포인트 도출

### Step 5: 성공 지표 정의

AI가 물어보는 질문들:
- 성공을 어떻게 측정할 건가요?
- KPI(핵심 성과 지표)는?
- 목표 수치는?

### Step 6: 제약 조건 및 가정

정리하는 내용:
- 기술적 제약
- 비즈니스 제약
- 전제 조건들

### Step 7: PRD 문서 생성

모든 정보를 종합하여 `prd.md` 파일 생성

---

## AI의 대화 스타일

### 페르소나
- **역할**: 스타트업 전문 투자자
- **톤**: 날카롭지만 건설적인 피드백
- **목표**: 비개발자가 이해할 수 있는 수준으로 안내

### 질문 방식
```
다음 중 가장 적합한 것을 선택해주세요:

1. [옵션 A] (추천)
2. [옵션 B]
3. [옵션 C]
4. [옵션 D]
5. 직접 입력
```

### 피드백 예시
- "좋은 아이디어네요. 하지만 타겟 시장이 너무 넓어 보여요."
- "경쟁사 분석 결과, 이 부분에서 차별화가 필요해 보입니다."

## 출력 문서 구조 (prd.md)

```markdown
# [프로젝트명] PRD

## 1. 개요
- 프로젝트 설명
- 해결하려는 문제

## 2. 목표 사용자
- 타겟 페르소나
- 사용자 니즈

## 3. 핵심 기능
- MVP 필수 기능
- 향후 추가 기능

## 4. 사용자 스토리
- 기능별 사용자 스토리

## 5. 경쟁사 분석
- 주요 경쟁사
- 차별화 포인트

## 6. 성공 지표
- KPI 정의
- 목표 수치

## 7. 제약 조건
- 기술적 제약
- 비즈니스 제약
```
