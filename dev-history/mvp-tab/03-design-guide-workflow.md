# Design Guide (UI 디자인 가이드) 워크플로우

## 개요

Design Guide 워크플로우는 UX Design을 바탕으로 시각적 디자인 가이드라인을 만드는 세 번째 단계입니다. 색상, 폰트, 컴포넌트 라이브러리 등을 결정합니다.

## 파일 위치
- **프롬프트 파일**: `src/constants/workflows/planning/startup-ui.ts`
- **출력 파일**: `anyon-docs/planning/design-guide.md`
- **참조 파일**: PRD, UX Design

---

## 워크플로우 진행 순서

### Step 1: 이전 문서 분석

AI가 수행하는 작업:
- PRD의 브랜드 톤 파악
- UX Design의 화면 구조 파악
- 타겟 사용자의 선호도 추론

### Step 2: 디자인 방향 결정

AI가 물어보는 질문들:
- 앱의 전체적인 느낌은? (모던/클래식/미니멀 등)
- 주요 색상 톤은? (밝은/어두운/화려한 등)
- 참고하고 싶은 앱이나 웹사이트가 있나요?

### Step 3: 컴포넌트 라이브러리 선택 (WebSearch 사용)

**중요**: AI가 웹 검색으로 최신 UI 라이브러리를 조사합니다.

조사 대상:
- shadcn/ui
- Tailwind CSS
- Material UI
- Ant Design
- 기타 최신 라이브러리

결정 기준:
- 프로젝트 요구사항과의 적합성
- 커스터마이징 용이성
- 커뮤니티 지원

### Step 4: 색상 팔레트 정의

정의하는 내용:
- Primary Color (주요 색상)
- Secondary Color (보조 색상)
- Background Color (배경 색상)
- Text Color (텍스트 색상)
- Accent Color (강조 색상)

### Step 5: 타이포그래피 정의

정의하는 내용:
- 폰트 패밀리
- 제목 크기 (H1, H2, H3...)
- 본문 크기
- 줄 간격

### Step 6: 공통 컴포넌트 정의

정의하는 컴포넌트:
- 버튼 스타일
- 입력 필드 스타일
- 카드 스타일
- 모달/팝업 스타일
- 네비게이션 스타일

### Step 7: Design Guide 문서 생성

모든 정보를 종합하여 `design-guide.md` 파일 생성

---

## AI의 대화 스타일

### 페르소나
- **역할**: UI/Visual 디자이너
- **접근법**: 트렌드 반영 + 사용성 고려
- **목표**: 일관된 시각적 경험 제공

### 추천 방식
```
이 프로젝트에 적합한 UI 라이브러리를 조사했습니다:

1. shadcn/ui (추천)
   - 장점: 커스터마이징 자유도 높음
   - 단점: 직접 설정 필요

2. Material UI
   - 장점: 풍부한 컴포넌트
   - 단점: 번들 사이즈 큼

어떤 것을 선택하시겠어요?
```

## 핵심 개념 설명

### 디자인 시스템
앱 전체에서 일관되게 사용되는 디자인 규칙과 컴포넌트의 모음입니다.

### 컴포넌트 라이브러리
미리 만들어진 UI 부품들의 모음입니다. 버튼, 입력창 등을 직접 만들지 않고 가져다 쓸 수 있습니다.

### 색상 팔레트
앱에서 사용할 색상들의 조합입니다. 보통 주요 색상 1-2개, 보조 색상 2-3개로 구성합니다.
