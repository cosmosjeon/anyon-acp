# PRD (Product Requirements Document) 워크플로우

## 개요

PRD 워크플로우는 **비개발자 창업자**의 아이디어를 체계적인 제품 요구사항 문서로 변환하는 **첫 번째** 단계입니다.

### 이 워크플로우가 하는 일

1. 창업자와 대화하며 아이디어를 구체화
2. 투자자 관점의 날카로운 질문으로 빈틈 점검
3. WebSearch로 경쟁사 자동 분석
4. 모든 정보를 정형화된 PRD 문서로 생성

---

## 실행 조건

### 시작 조건
- 사용자가 기획문서 탭에서 "PRD 작성 시작" 버튼 클릭
- 프로젝트 폴더가 선택되어 있어야 함

### 종료 조건
- `anyon-docs/planning/prd.md` 파일 생성 완료
- 자동으로 UX Design 탭 활성화

---

## 워크플로우 상세 단계

### Step 1: 서비스 유형 파악

AI가 처음 묻는 핵심 질문:

```
어떤 서비스를 만들려고 하시나요? 다음 중 선택해주세요:

1. SaaS (구독형 웹 서비스) - 예: Notion, Figma (추천)
2. 이커머스/마켓플레이스 - 예: 쿠팡, 당근마켓
3. 콘텐츠/미디어 플랫폼 - 예: 유튜브, 브런치
4. 사내 업무용 도구 - 예: ERP, 그룹웨어
5. 모바일 앱 서비스 - 예: 배달의민족, 토스
6. 기타 (직접 입력)
```

**AI 판단 로직**:
- 사용자 선택에 따라 이후 질문 분기
- SaaS → 구독 모델 관련 질문 추가
- 이커머스 → 결제/물류 관련 질문 추가

### Step 2: 핵심 문제 정의

```
이 서비스가 해결하려는 핵심 문제는 무엇인가요?

1. 기존 방식이 너무 복잡해서 단순화가 필요 (추천)
2. 비용이 너무 비싸서 저렴한 대안 필요
3. 기존 솔루션이 아예 없어서 새로 만들어야 함
4. 기존 방식이 너무 느려서 속도 개선 필요
5. 특정 그룹이 소외되어 있어서 맞춤 서비스 필요
6. 기타 (직접 입력)
```

**AI 판단 로직**:
- "비용" 선택 시 → 가격 경쟁력 질문 추가
- "속도" 선택 시 → 기술적 차별화 질문 추가

### Step 3: 타겟 사용자 정의

```
주요 타겟 사용자는 누구인가요?

1. B2C - 일반 소비자 개인
2. B2B - 기업 고객 (스타트업/SMB)
3. B2B - 기업 고객 (대기업/엔터프라이즈)
4. B2B2C - 기업을 통해 개인에게
5. 프리랜서/1인 기업
6. 기타 (직접 입력)
```

**AI 판단 로직**:
- B2B 선택 시 → 도입 결정자(구매자)와 실제 사용자 구분 질문
- B2C 선택 시 → 연령대, 성별, 관심사 등 페르소나 질문

### Step 4: 경쟁사 분석 (WebSearch 자동 실행)

**중요**: 이 단계에서 AI가 **자동으로 WebSearch 도구를 사용**합니다.

```
[AI 내부 동작]
1. 사용자가 입력한 서비스 유형, 문제, 타겟을 조합
2. 검색어 생성: "{서비스유형} {문제} 서비스 경쟁사"
3. WebSearch 실행
4. 검색 결과에서 경쟁사 5-10개 추출
5. 각 경쟁사의 강점/약점 분석
6. 차별화 포인트 제안
```

**사용자에게 보여지는 결과**:
```
경쟁사 분석 결과입니다:

| 경쟁사 | 강점 | 약점 | 가격 |
|-------|------|-----|------|
| A사 | 사용 편의성 | 기능 제한적 | 월 $10 |
| B사 | 기능 다양 | 복잡한 UI | 월 $30 |
| C사 | 무료 | 광고 많음 | 무료 |

→ 추천 차별화 포인트: [AI가 분석한 내용]
```

### Step 5: 핵심 기능 정의

```
핵심 기능을 선택해주세요 (복수 선택 가능):

1. 사용자 인증/로그인 (추천)
2. 대시보드/관리 화면
3. 결제/구독 시스템
4. 실시간 알림
5. 데이터 분석/리포트
6. 외부 서비스 연동 (API)
7. 협업/공유 기능
8. 기타 (직접 입력)
```

**AI 판단 로직**:
- 선택된 기능들을 MVP 필수 / 2차 개발로 자동 분류
- 기능 간 의존성 분석 (예: 결제 → 사용자 인증 필수)

### Step 6: MVP 범위 확정

```
MVP(최소 기능 제품)에 반드시 포함할 기능을 3개까지 선택해주세요:

[Step 5에서 선택한 기능 목록 중 선택]

(선택하지 않은 기능은 2차 개발로 분류됩니다)
```

### Step 7: PRD 문서 생성

모든 정보를 종합하여 `prd.md` 파일 자동 생성

## AI 페르소나 상세

### 역할
**스타트업 투자 경험이 있는 비즈니스 전문가**

### 대화 원칙

1. **날카롭지만 건설적**
   - "좋은 아이디어네요. 하지만 타겟 시장이 너무 넓어 보여요. 좁혀볼까요?"
   - "경쟁사 분석 결과, 이 부분에서 차별화가 필요해 보입니다."

2. **구체적인 수치와 사례 제시**
   - 모호한 표현 금지 ("더 좋은", "편리한" 등)
   - 항상 비교 대상이나 수치 포함

3. **창업자가 놓친 부분 짚어주기**
   - "이 기능은 구현 난이도가 높은데, MVP에 꼭 필요할까요?"
   - "타겟 사용자가 실제로 돈을 낼 의향이 있을까요?"

### 질문 형식 규칙

```yaml
question_rules:
  min_options: 4         # 최소 4개 선택지
  max_options: 8         # 최대 8개 선택지
  allow_multiple: true   # 복수 선택 허용
  show_recommendation: true  # (추천) 표시
  always_include_other: true # 항상 "기타" 포함
```
