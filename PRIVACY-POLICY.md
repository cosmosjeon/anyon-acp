**주식회사 SLIT** (이하 "회사")는 「개인정보 보호법」 제30조에 따라 정보주체의 개인정보를 보호하고 이와 관련한 고충을 신속하고 원활하게 처리할 수 있도록 하기 위하여 다음과 같이 개인정보 처리방침을 수립·공개합니다.

**시행일자**: 2025년 12월 25일

## 2. 처리하는 개인정보의 항목

회사는 다음의 개인정보 항목을 처리하고 있습니다.

### 2.1 필수 항목 (Google OAuth 로그인)
- **이메일 주소**: 회원 식별 및 로그인
- **이름**: 서비스 내 사용자 식별
- **프로필 사진 URL**: 사용자 인터페이스 표시
- **Google ID**: 계정 연동 및 중복 방지

### 2.2 서비스 제공 과정에서 자동 생성되는 정보
- **사용자 고유 식별자 (UUID)**: 내부 시스템 관리
- **구독 플랜 정보**: FREE/PRO 플랜 구분
- **구독 상태**: ACTIVE/CANCELED/PAST_DUE
- **구독 기간 종료일**: 결제 주기 관리
- **사용자 설정**: 애플리케이션 환경 설정 (키-값 쌍)
- **API 사용량 정보**: 일일 토큰 사용량 및 비용
- **접속 로그**: 접속 일시, IP 주소

### 2.3 선택적 수집 정보 (분석 서비스 동의 시)
- **익명 사용자 ID**: 통계 분석용 (개인 식별 불가)
- **세션 ID**: 세션 추적용
- **이벤트 데이터**: 사용자 행동 패턴 분석 (버튼 클릭, 페이지 이동 등)

**※ 회사는 사용자의 민감정보(인종, 사상·신념, 정치적 성향, 건강, 성생활 등)를 수집하지 않습니다.**

- **보유 및 이용기간**: Anthropic 개인정보 처리방침에 따름
- **거부권**: 프록시 대신 직접 API 키 사용 가능

### 4.3 Google LLC (Gemini AI)
- **제공 목적**: AI 기반 고객 지원 채팅
- **제공 항목**: 사용자가 입력한 지원 채팅 메시지
- **보유 및 이용기간**: Google 개인정보 처리방침에 따름

### 4.4 PostHog Inc. (선택적 동의 시)
- **제공 목적**: 서비스 이용 통계 및 분석

## 7. 개인정보의 파기 절차 및 방법

회사는 개인정보 보유기간의 경과, 처리목적 달성 등 개인정보가 불필요하게 되었을 때에는 지체 없이 해당 개인정보를 파기합니다.

### 7.1 파기 절차

### 7.2 파기 방법
- **전자적 파일 형태**: 복구 및 재생이 불가능한 기술적 방법을 사용하여 완전히 삭제

- 앱 내 설정 → 계정 관리 → 회원 탈퇴
- 탈퇴 즉시 모든 개인정보 파기 (법령 보존 의무 제외)

## 9. 개인정보의 안전성 확보 조치

회사는 「개인정보 보호법」 제29조에 따라 다음과 같이 안전성 확보에 필요한 기술적·관리적·물리적 조치를 하고 있습니다.

### 9.1 기술적 조치
1. **암호화 통신**
   - HTTPS(TLS 1.2 이상) 프로토콜을 통한 데이터 전송 암호화
   - 모든 API 통신 암호화 적용

2. **인증 및 접근 제어**
   - Google OAuth 2.0 기반 인증
   - JWT(JSON Web Token) 토큰 기반 세션 관리 (7일 만료)
   - 토큰 블랙리스트를 통한 로그아웃 처리

3. **민감 정보 보호**
   - API 키는 시스템 키체인/자격증명 관리자에 암호화 저장
   - macOS: Security Keychain
   - Windows: Credential Manager
   - Linux: secret-tool 또는 암호화된 파일

4. **Rate Limiting (속도 제한)**
   - 인증 API: 100회/15분 (IP 기준)
   - 개발 API: 20회/15분 (IP 기준)
   - 무차별 대입 공격(Brute Force) 방지

5. **CORS 화이트리스트**
   - 허용된 출처(Origin)에서만 API 접근 가능
   - localhost, auth.any-on.com, tauri:// 프로토콜만 허용

### 9.2 관리적 조치
1. **내부관리계획 수립 및 시행**
   - 개인정보 처리 관련 안정성 확보를 위한 내부관리계획 수립 및 시행

2. **개인정보 취급 직원의 최소화 및 교육**
   - 개인정보를 취급하는 직원을 지정하고 최소화
   - 담당자에 한정시켜 관리

3. **정기적인 자체 점검**
   - 개인정보 취급 관련 안정성 확보를 위해 정기적으로 자체 점검 실시

### 9.3 물리적 조치
1. **데이터베이스 보안**
   - SQLite WAL(Write-Ahead Logging) 모드 적용
   - Foreign Key 제약조건 적용
   - Prepared Statements를 통한 SQL Injection 방지

2. **접근 통제**
   - 개인정보를 보관하는 데이터베이스 시스템에 대한 접근권한 부여·변경·말소 관리
   - 침입차단시스템을 이용한 외부로부터의 무단 접근 통제

## 11. 쿠키(Cookie) 및 자동 수집 장치

### 11.1 쿠키의 사용
회사는 개인화되고 맞춤화된 서비스를 제공하기 위해 이용자의 정보를 저장하고 수시로 불러오는 '쿠키(cookie)'를 사용합니다.

### 11.2 쿠키란?
쿠키는 웹사이트를 운영하는데 이용되는 서버가 이용자의 브라우저에 보내는 아주 작은 텍스트 파일로서 이용자 컴퓨터에 저장됩니다.

### 11.3 회사의 쿠키 사용 목적
- **로그인 세션 유지**: JWT 토큰 저장 (httpOnly, secure 플래그 적용)
- **보안**: CSRF 방어

### 11.4 쿠키 설치·운영 및 거부 방법
이용자는 쿠키 설치에 대한 선택권을 가지고 있습니다. 웹브라우저에서 옵션을 설정함으로써 모든 쿠키를 허용하거나, 쿠키가 저장될 때마다 확인을 거치거나, 모든 쿠키의 저장을 거부할 수 있습니다.

**※ 단, 쿠키 설치를 거부할 경우 로그인이 필요한 일부 서비스 이용에 어려움이 있을 수 있습니다.**

### 11.5 localStorage 사용
앱은 브라우저 localStorage를 사용하여 다음 정보를 저장합니다:
- 인증 상태 (`auth-storage`)
- 분석 동의 여부 (`anyon-analytics-consent`)
- 애플리케이션 설정 (`app_setting:*`)
- 지원 채팅 히스토리
