# TRD 워크플로우 예시

펫시터 매칭 앱 "펫케어" 프로젝트의 TRD 워크플로우 진행 예시입니다.

---

## 예시 1: 오픈소스 없이 처음부터

### Step 0: 문서 로드 (자동)

AI가 PRD, UX Design, Design Guide를 자동으로 로드합니다.

**Claude:**
```
펫케어의 기술 스택을 정할게요!

몇 가지만 확인하면 나머지는 제가 알아서 정할게요.
```

---

### Step 1: 예산 질문

**Claude:**
```
월 인프라 예산은 어느 정도인가요?

1. **무료** - 테스트/MVP용
   - 적합: 사용자 100명 이하
   - 일부 기능 제한 있음

2. **소규모 (1-5만원)** - 소규모 실서비스
   - 적합: 사용자 1,000명 이하

3. **중규모 (5-20만원)** - 안정적 운영
   - 적합: 사용자 10,000명 이하

4. **대규모 (20만원+)** - 본격 서비스
   - 적합: 사용자 10,000명 이상

번호로 선택하거나, 궁금한 점 물어보세요!
```

**User:**
```
1번이요. 일단 MVP라서요.
```

---

### Step 2: 서버 구성 질문

**Claude:**
```
서버를 어떻게 구성할까요?

1. **올인원 서비스 사용** (Supabase, Firebase 등)
   - 빠르게 만들 수 있어요
   - 데이터베이스, 로그인, 파일 저장이 한 번에 해결돼요
   - 나중에 복잡한 기능 추가가 어려울 수 있어요

2. **직접 만들기**
   - 원하는 대로 자유롭게 만들 수 있어요
   - 개발 시간이 더 걸려요
   - 장기적으로 유연해요

추천: 1번 (MVP라서 빠르게 만드는 게 좋아요)

번호로 선택하거나, 궁금한 점 물어보세요!
```

**User:**
```
1번이요
```

---

### Step 3: UI Library 질문

**Claude:**
```
디자인 가이드를 봤을 때, 이 UI 라이브러리들이 잘 맞을 것 같아요:

1. **shadcn/ui** - 커스터마이징 자유도 높음 (추천)
   - 디자인 수정하기 가장 쉬워요
   - 친근한 스타일에 잘 맞아요

2. **MUI** - 완성도 높은 컴포넌트
   - 빠르게 만들 수 있어요

3. **Chakra UI** - 접근성 중심
   - 깔끔하고 사용하기 쉬워요

번호로 선택하거나, 궁금한 점 물어보세요!
```

**User:**
```
1번이랑 2번 차이가 뭐야?
```

**Claude:**
```
좋은 질문이에요!

**shadcn/ui:**
- 컴포넌트 코드를 복사해서 직접 수정해요
- 디자인 가이드에 맞게 색상/스타일 바꾸기 쉬워요
- 번들 크기가 작아요

**MUI:**
- 이미 완성된 컴포넌트를 가져다 써요
- 커스터마이징하려면 테마 설정이 복잡해요
- 컴포넌트 종류가 더 많아요

펫케어는 친근한 디자인이라 커스터마이징이 중요해서
1번 shadcn/ui를 추천드려요!
```

**User:**
```
그럼 1번으로 할게요
```

---

### Step 4: AI 자동 결정 (웹검색)

AI가 웹검색으로 나머지 기술을 결정합니다.

**Claude:**
```

| 분류 | 선택 |
|-----|-----|
| 화면 | Next.js |
| UI | shadcn/ui (선택하신 거) |
| 서버 | Supabase |
| 데이터 저장 | PostgreSQL (Supabase 내장) |
| 호스팅 | Vercel |

**추가 서비스:**
- 인증: Supabase Auth (소셜 로그인 포함)
- 파일 저장: Supabase Storage (펫/펫시터 사진)

다음 단계로 넘어갈까요? 조정하고 싶은 부분이 있으면 말씀해주세요!
```

**User:**
```
좋아요!
```

## 예시 2: 오픈소스 활용 (완성형)

PRD에서 cal.com을 기반으로 예약 시스템을 만들기로 한 경우입니다.

### Step 0: 문서 로드 + 오픈소스 클론

**Claude:**
```
예약 서비스의 기술 스택을 정할게요!

**기반 오픈소스**: cal.com
**클론 위치**: /cal.com

코드베이스를 분석해봤어요:
- Frontend: Next.js 14, React
- Backend: tRPC, Prisma
- Database: PostgreSQL
- 인증: NextAuth

기반 프로젝트의 기술 스택을 그대로 사용하면서
필요한 부분만 수정하면 될 것 같아요!

몇 가지만 확인할게요.
```

### Step 1: 예산 질문

**Claude:**
```
월 인프라 예산은 어느 정도인가요?

1. **무료** - 테스트/MVP용
2. **소규모 (1-5만원)** - 소규모 실서비스
3. **중규모 (5-20만원)** - 안정적 운영
4. **대규모 (20만원+)** - 본격 서비스

번호로 선택하거나, 궁금한 점 물어보세요!
```

**User:**
```
2번이요
```

### Step 2: 서버 구성 질문 (스킵)

> cal.com이 이미 백엔드를 포함하므로 이 질문은 스킵됩니다.

### Step 3: UI Library 질문

**Claude:**
```
cal.com은 이미 자체 UI 컴포넌트를 가지고 있어요.

1. **기존 유지** - cal.com 컴포넌트 그대로 사용 (추천)
   - 가장 빠르고 안정적이에요

2. **일부 교체** - shadcn/ui로 일부 컴포넌트만 교체
   - 디자인 커스터마이징 필요할 때

3. **전체 교체** - 다른 UI 라이브러리로 전체 교체
   - 시간이 많이 걸려요

번호로 선택하거나, 궁금한 점 물어보세요!
```

**User:**
```
1번으로 할게요. 디자인은 색상만 바꾸면 될 것 같아요.

---

# 펫케어 - TRD (기술 요구사항)

## 예산

| 항목 | 내용 |
|-----|------|
| 월 예산 | 무료 |
| 예상 비용 | $0/월 (무료 tier) |

---

## 기술 스택

| 분류 | 기술 | 비고 |
|-----|-----|------|
| Frontend | Next.js | React 기반 SSR |
| UI Library | shadcn/ui | 사용자 선택 |
| Backend | Supabase | 올인원 서비스 |
| Database | PostgreSQL | Supabase 내장 |
| Hosting | Vercel | 무료 tier |

---

## 추가 서비스

| 서비스 | 선택 | 비고 |
|-------|-----|------|
| 인증 | Supabase Auth | 소셜 로그인 포함 |
| 파일 저장 | Supabase Storage | 1GB 무료 |

---

## 비용 예측

### MVP (0-100 사용자)
- Vercel Free: $0
- Supabase Free: $0
- **총: $0/월**

### 성장기 (100-1,000 사용자)
- Vercel Pro: $20/월
- Supabase Pro: $25/월
- **총: ~$45/월 (약 6만원)**
```

---

## 핵심 포인트

1. **질문 3개**: 예산, 서버구성, UI Library
2. **자연스러운 대화**: "차이가 뭐야?" 같은 추가 질문 OK
3. **AI 자동 결정**: 나머지 기술은 웹검색으로 결정
4. **오픈소스 클론**: 있으면 루트에 클론하고 분석
5. **비개발자 언어**: "올인원 서비스", "직접 만들기"
