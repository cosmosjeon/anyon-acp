# UX Design 워크플로우

## 개요

UX Design 워크플로우는 PRD를 바탕으로 **유저 플로우 중심의 대화**를 통해 화면을 설계하는 단계입니다.

## 파일 위치
- **프롬프트 파일**: `src/constants/workflows/planning/startup-ux.ts`
- **출력 파일**: `anyon-docs/planning/ui-ux.html`

---

## 핵심 원칙

1. **핵심 비즈니스 플로우에만 집중** - 사용자와 논의할 건 핵심 로직만
2. **엣지케이스는 AI가 알아서** - 온보딩, 에러 처리, 로그인 등은 AI가 자동 추가
3. **열린 질문 → 상세 작성 요청** - 사용자가 직접 플로우를 적게
4. **대화로 구체화** - 적은 내용 바탕으로 깊이 있는 논의
5. **화면 리뷰 반복** - HTML 생성 후 사용자 피드백으로 수정

---

## AI 페르소나

**역할:** 유저 플로우를 함께 그려가는 UX 파트너

- 유저 플로우 개념을 쉽게 설명
- 사용자가 상세하게 적도록 유도
- 핵심 비즈니스 로직에 대해서만 질문
- 엣지케이스는 알아서 처리

---

## 사용자와 논의할 것 vs 안 할 것

| 논의 O (핵심 비즈니스) | 논의 X (AI가 처리) |
|------------------------|-------------------|
| 메인 사용 플로우 | 온보딩/로그인 |
| 핵심 기능의 세부 UX | 에러/빈 상태 |
| 차별화 포인트 | 설정/프로필 |
| 핵심 의사결정 기준 | 일반적인 엣지케이스 |

---

## 워크플로우 진행 순서 (6단계)

### Step 1: PRD 읽기 + 유저 플로우 개념 설명

```
"안녕하세요! {{project_name}}의 화면 설계를 시작할게요.

먼저 '유저 플로우'가 뭔지 간단히 설명드릴게요.

**유저 플로우란?**
사용자가 서비스에서 목표를 달성하기까지의 과정이에요.

예를 들어 '배달의민족'에서 음식 주문하려면:
앱 열기 → 음식 고르기 → 장바구니 담기 → 결제하기 → 주문 완료

이렇게 '시작부터 목표 달성까지'의 흐름이에요."
```

### Step 2: 핵심 비즈니스 플로우 질문 (열린 질문)

```
"{{project_name}}에서 가장 핵심적인 사용자 행동은 뭘까요?

예를 들어:
- 쇼핑몰이면 '상품 검색 → 장바구니 → 결제'
- SNS면 '피드 보기 → 좋아요/댓글 → 글 작성'

생각하시는 핵심 플로우가 있으면 **최대한 상세하게** 적어주세요!
구체적일수록 좋아요.

예시:
'날짜/위치 입력 → 조건에 맞는 펫시터 목록 → 프로필에서 검증뱃지랑 리뷰 확인
→ 예약 날짜/시간 선택 → 결제 → 돌봄 시작되면 실시간 사진 알림으로 받기'"
```

→ 사용자가 상세하게 작성하도록 유도
→ 없으면 PRD 기반으로 함께 만들어가기

### Step 3: 핵심 플로우 구체화 대화

사용자가 적은 플로우의 **비즈니스 로직**에 대해서만 질문:

```
"좋아요! 정리해보면:
[사용자가 적은 플로우 정리]

비즈니스적으로 중요한 부분 몇 가지 여쭤볼게요:

1. [핵심 화면]에서 가장 먼저 눈에 띄어야 할 건 뭘까요?
2. [핵심 기능]은 어떤 형태면 좋을까요?

또는 다르게 생각하시는 게 있으면 말씀해주세요!"
```

**질문 안 할 것 (AI가 처리):**
- 첫 사용자 온보딩
- 회원가입/로그인 플로우
- 에러 상태
- 빈 화면 상태
- 설정/프로필

### Step 4: 최종 정리 + HTML 생성

```
"정리할게요!

**메인 유저 플로우:**
[대화에서 나온 핵심 비즈니스 플로우]

이걸 바탕으로 화면을 만들게요.
(회원가입, 로그인, 설정 등 기본 화면은 제가 알아서 추가할게요!)

진행할까요?"
```

→ 확인 후 HTML 와이어프레임 생성
→ 엣지케이스/시스템 화면은 AI가 자동 추가

### Step 5: 화면 리뷰 + 수정 (반복)

```
"화면이 만들어졌어요!

브라우저에서 열어서 클릭해보시고,
**수정하고 싶은 부분**이 있으면 말씀해주세요!

예시:
- '검색 화면에서 필터가 더 눈에 띄었으면 좋겠어요'
- '프로필 화면에 가격이 더 크게 보였으면'
- '결제 전에 확인 단계가 하나 더 있으면 좋겠어요'
"
```

→ 사용자 피드백 받음
→ 수정 반영
→ 다시 확인 요청
→ 만족할 때까지 반복

### Step 6: 완료 안내

```
"수정이 완료되었어요!

화면 구성이 마음에 드시면,
**'디자인가이드 작성하기'** 버튼을 눌러주세요.

다음 단계에서는 색상, 폰트, 간격 등
실제 디자인을 정하게 됩니다."
```

## AI가 자동으로 추가하는 것들

사용자와 논의 없이 AI가 알아서 만드는 화면/플로우:
- 온보딩 (스플래시, 앱 소개)
- 회원가입/로그인
- 비밀번호 찾기
- 프로필/설정
- 에러 상태
- 빈 화면 상태 (데이터 없음)
- 로딩 상태

---

## 출력물: HTML 와이어프레임

### 특징
- 브라우저에서 바로 열 수 있는 단일 HTML 파일
- 모든 버튼이 클릭 가능 (화면 전환)
- 그레이스케일 스타일 (구조와 흐름에 집중)
- 이모지 없음

### 스타일 규칙
- 색상: 그레이스케일만 (#000, #fff, #333, #666, #999, #ccc, #eee)
- 폰트: 시스템 기본 폰트
- 테두리: 1px solid
- 장식 요소 없음
