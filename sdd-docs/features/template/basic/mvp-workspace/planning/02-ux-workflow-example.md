# UX Design 워크플로우 예시 (펫시터 매칭 서비스)

## 예시 대화 흐름

### Step 1: 유저 플로우 개념 설명

**Claude:**
```
안녕하세요! 펫시터 매칭 서비스의 화면 설계를 시작할게요.

먼저 '유저 플로우'가 뭔지 간단히 설명드릴게요.

**유저 플로우란?**
사용자가 서비스에서 목표를 달성하기까지의 과정이에요.

예를 들어 '배달의민족'에서 음식 주문하려면:
앱 열기 → 음식 고르기 → 장바구니 담기 → 결제하기 → 주문 완료

이렇게 '시작부터 목표 달성까지'의 흐름이에요.

---

**PRD에서 가져온 정보:**
- 프로젝트: 펫시터 매칭 서비스
- 서비스 유형: 모바일 앱
- 플랫폼: iOS/Android
- 타겟 사용자: 출장이 잦은 직장인

**핵심 기능:**
1. 펫시터 검증 시스템
2. 실시간 사진 리포트
3. 날짜/위치 기반 검색
4. 결제/정산 시스템
```

---

### Step 2: 핵심 플로우 질문 (열린 질문)

**Claude:**
```
펫시터 매칭 서비스에서 가장 핵심적인 사용자 행동은 뭘까요?

예를 들어:
- 쇼핑몰이면 '상품 검색 → 장바구니 → 결제'
- SNS면 '피드 보기 → 좋아요/댓글 → 글 작성'

생각하시는 핵심 플로우가 있으면 **최대한 상세하게** 적어주세요!
구체적일수록 좋아요.

**예시:**
'날짜/위치 입력 → 조건에 맞는 펫시터 목록 → 프로필에서 검증뱃지랑 리뷰 확인
→ 예약 날짜/시간 선택 → 결제 → 돌봄 시작되면 실시간 사진 알림으로 받기'

---

아직 구체적인 생각이 없으시면 '없어요'라고 하셔도 괜찮아요!
PRD 기반으로 함께 만들어갈 수 있어요.
```

**User (패턴 1: 상세하게 작성):**
```
날짜랑 위치 입력해서 조건에 맞는 펫시터 검색하고,
프로필 들어가서 검증 뱃지랑 리뷰 확인하고,
마음에 들면 예약 날짜 선택해서 결제하고,
돌봄 시작되면 펫시터가 보내는 사진을 푸시 알림으로 받아보는 거요.
사진 누르면 크게 볼 수 있고, 그날 받은 사진들 모아보기도 되면 좋겠어요.
```

**또는 User (패턴 2: 위임):**
```
잘 모르겠어요, 알아서 만들어주세요
```

**→ AI 응답 (위임 시):**
```
PRD를 바탕으로 핵심 플로우를 정리해봤어요:

검색(날짜/위치) → 펫시터 목록 → 프로필(검증뱃지+리뷰) → 예약/결제 → 실시간 사진 리포트

이 플로우로 화면을 만들까요? 수정하고 싶은 부분 있으면 말씀해주세요!
```

---

### Step 3: 핵심 비즈니스 논의

**Claude:**
```
좋아요! 정리하면:

**메인 유저 플로우:**
검색(날짜/위치) → 펫시터 목록 → 프로필(검증+리뷰) → 예약/결제 → 실시간 사진 리포트

비즈니스적으로 중요한 부분 여쭤볼게요:

펫시터 프로필에서 가장 먼저 눈에 띄어야 할 건 뭘까요?

1. 검증 배지 (신분증, 범죄이력 확인됨) - 믿을 수 있는지 바로 확인
2. 별점 + 리뷰 개수 - 다른 사람들 후기
3. 경력/자기소개 - 펫시터가 어떤 사람인지
4. 시간당 가격 - 예산에 맞는지

또는 다르게 생각하시는 게 있으면 말씀해주세요!
```

**User:** `1이랑 2요. 믿을 수 있는지가 제일 중요해서`

**Claude:**
```
그럼 프로필 상단에 검증 배지 + 평점/리뷰를 크게 배치할게요.

사진 리포트는 어떤 느낌이면 좋을까요?

1. 카톡 채팅방처럼 - 사진 + 간단한 메시지 ("밥 잘 먹었어요~")
2. 인스타 피드처럼 - 사진 중심 갤러리
3. 타임라인 - 시간순 기록 (10:00 산책, 12:00 밥)

또는 다른 느낌이 있으시면요?
```

**User:** `1번이요. 펫시터가 "밥 잘 먹었어요~" 이런 거 같이 보내주면 좋을 것 같아서`

---

### Step 4: 최종 정리 + HTML 생성

**Claude:**
```
정리할게요!

**메인 유저 플로우:**
검색 → 펫시터 목록 → 프로필(검증+리뷰 강조) → 예약/결제 → 채팅형 사진 리포트

**핵심 UX 결정사항:**
- 프로필 상단: 검증 배지 + 평점/리뷰 크게
- 사진 리포트: 카톡 채팅방 스타일 (사진 + 메시지)

이걸 바탕으로 화면을 만들게요.
(로그인, 설정 등 기본 화면은 제가 알아서 추가할게요!)

진행할까요?
```

**User:** `네!`

### Step 6: 화면 리뷰 + 수정

**Claude:**
```
화면이 만들어졌어요!

브라우저에서 열어서 클릭해보시고,
수정하고 싶은 부분이 있으면 말씀해주세요!
```

**User:**
```
펫시터 프로필 화면에서 예약 버튼이 너무 아래에 있어요.
스크롤 안 해도 바로 보이게 하단에 고정되면 좋겠어요.
```

**Claude:**
```
예약 버튼을 하단에 고정할게요!
수정했어요. 다시 확인해보세요.
```

**User:**
```
좋아요! 근데 검색 결과 화면에서 펫시터 카드에 가격도 바로 보이면 좋겠어요.
```

**Claude:**
```
펫시터 카드에 가격 추가할게요.
수정 완료! 다른 수정할 부분 있으세요?
```

**User:** `없어요 이제 괜찮아요!`

## 워크플로우 특징

### 이전 방식 vs 새 방식

| 이전 (8단계) | 새 방식 (6단계) |
|-------------|----------------|
| AI가 화면 목록 제안 | 사용자가 플로우 직접 작성 |
| "1" 선택만 | 자유롭게 상세 작성 |
| 화면 → 연결 순서 | 플로우 → 화면 순서 |
| 엣지케이스도 사용자 확인 | 엣지케이스는 AI가 자동 |
| 기술 용어 (네비게이션, 와이어프레임) | 쉬운 말 (화면, 흐름) |
| HTML 생성 후 끝 | HTML 생성 후 리뷰+수정 반복 |

### 사용자와 논의 vs AI가 처리

| 논의 O (핵심 비즈니스) | 논의 X (AI가 처리) |
|------------------------|-------------------|
| 메인 사용 플로우 | 온보딩/로그인 |
| 프로필에서 강조할 정보 | 에러/빈 상태 |
| 사진 리포트 형태 | 설정/프로필 |
| 핵심 UX 결정 | 시스템 화면들 |
