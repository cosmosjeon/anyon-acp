# PRD Validation Checklist

## 📋 문서 구조

- [ ] YAML frontmatter가 모든 필수 메타데이터를 포함하고 있음
- [ ] document_type, project_name, service_type, platform이 명시됨
- [ ] core_features와 success_metrics가 리스트 형태로 정리됨
- [ ] 모든 섹션이 존재하고 placeholder가 없음

## 🎯 프로젝트 비전

- [ ] 해결하고자 하는 문제가 구체적으로 정의됨
- [ ] 타겟 사용자가 명확하게 정의됨 (페르소나 수준)
- [ ] 차별화 포인트가 측정 가능하거나 명확하게 설명됨
- [ ] "왜 지금인가"가 시장/기술/트렌드 근거와 함께 설명됨

## 🔧 서비스 유형 및 플랫폼

- [ ] 서비스 유형이 명확히 선택됨 (웹/앱/설치형/하이브리드)
- [ ] 플랫폼이 구체적으로 정의됨 (PC/모바일/태블릿 등)
- [ ] 선택 이유가 타겟 사용자와 문제 해결 방식에 부합함
- [ ] 플랫폼 선택이 나중 UX/기술 결정에 충분한 가이드가 됨

## ⭐ 핵심 기능

- [ ] 3-7개의 핵심 기능이 정의됨
- [ ] 각 기능이 구체적으로 설명됨 (무엇을 하는지)
- [ ] 각 기능의 필요성이 명확함 (왜 필요한지)
- [ ] 각 기능에 사용 시나리오/예시가 포함됨
- [ ] 모든 기능이 정의된 문제 해결에 기여함
- [ ] 플랫폼 특성이 반영됨 (앱이면 푸시알림, 웹이면 반응형 등)

## 👥 사용자 페르소나

- [ ] 타겟 사용자의 특성이 구체적으로 묘사됨
- [ ] 사용자의 일상, 문제 상황, 니즈가 설명됨
- [ ] 기술 친숙도가 명시됨
- [ ] 사용자가 중요하게 생각하는 가치가 정의됨

## 💰 비즈니스 모델

- [ ] 수익 모델이 명확히 선택됨
- [ ] 구체적인 가격/수수료/광고 전략이 정의됨
- [ ] 타겟 사용자가 지불할 의향이 있는 가격대임
- [ ] 경쟁 서비스 대비 합리적인 수익 전략임

## 📊 성공 지표

- [ ] 측정 가능한 성공 지표가 정의됨
- [ ] 단기(3개월) 목표가 구체적인 숫자로 제시됨
- [ ] 중기(1년) 목표가 제시됨
- [ ] 지표가 비즈니스 모델과 일치함
- [ ] 달성 가능하면서도 의미 있는 수준의 목표임

## 🚀 MVP 범위

- [ ] MVP에 포함될 기능이 명확히 구분됨
- [ ] MVP가 핵심 문제를 해결할 수 있는 최소 기능만 포함함
- [ ] 향후 개발할 기능이 따로 정리됨
- [ ] MVP 범위가 현실적으로 개발 가능한 수준임

## 🎨 추가 요구사항

- [ ] UX/UI 방향성이 정의됨
- [ ] 보안 요구사항이 기능에 맞게 정의됨
- [ ] 접근성 요구사항이 고려됨
- [ ] 다국어/지역화 계획이 명시됨

## 🔄 일관성 검증

- [ ] 모든 핵심 기능이 문제 해결에 기여함
- [ ] 서비스 유형/플랫폼이 타겟 사용자에게 적합함
- [ ] 비즈니스 모델이 타겟 사용자의 지불 능력/의향에 맞음
- [ ] 성공 지표가 비즈니스 모델과 일치함

## ✅ AI 가독성

- [ ] 각 기능에 충분한 설명이 포함됨 (AI가 이해할 수준)
- [ ] 기술적 제약사항이 명시됨
- [ ] 다음 워크플로우(UX)에서 필요한 모든 정보가 포함됨
- [ ] YAML 메타데이터로 빠른 검색이 가능함

## 🚦 다음 단계 준비

- [ ] UX Design에 필요한 모든 기능이 정의됨
- [ ] 플랫폼과 서비스 유형이 명확해서 UX 설계 방향이 정해짐
- [ ] 타겟 사용자가 명확해서 페르소나 기반 UX 가능함
- [ ] 문서가 {default_output_file}에 저장됨

---

## 📝 검증 완료 후

모든 체크박스가 완료되면:
1. ✅ PRD 문서가 완성됨
2. ✅ UX Design 워크플로우로 넘어갈 준비 완료
3. ✅ startup-ux 워크플로우 자동 시작
