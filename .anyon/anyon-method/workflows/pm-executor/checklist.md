# PM Executor 검증 체크리스트

## 1. 사전 조건 검증

### 필수 입력 파일
- [ ] execution-plan.md 파일이 존재함 (pm-orchestrator가 생성)
- [ ] execution-plan.md에 최소 1개 이상의 Wave가 정의됨
- [ ] 각 Wave에 최소 1개 이상의 ticket이 포함됨

### 티켓 구조 검증
- [ ] 모든 티켓에 ticket_id가 있음
- [ ] 모든 티켓에 type이 지정됨
- [ ] 모든 티켓에 wave가 할당됨
- [ ] 모든 티켓에 acceptance_criteria가 있음

---

## 2. 실행 모드 검증

### 모드 선택
- [ ] 실행 모드가 interactive/semi-auto/full-auto 중 하나로 선택됨
- [ ] 프로그레스 파일이 초기화됨

---

## 3. Wave 실행 검증

### Wave 순서
- [ ] Wave 1부터 순차적으로 실행됨
- [ ] 각 Wave 시작 전 이전 Wave가 완료됨
- [ ] 병렬 실행 가능 티켓들이 올바르게 식별됨

### 의존성 준수
- [ ] blocked_by 티켓이 완료된 후에만 실행됨
- [ ] 순환 의존성이 발생하지 않음
- [ ] 의존성 해소 시 자동으로 상태가 업데이트됨

---

## 4. 티켓 실행 검증

### TDD 플로우
- [ ] RED: 테스트가 구현 전에 작성됨
- [ ] RED: 테스트가 초기에 실패함 (의도된 실패)
- [ ] GREEN: 테스트를 통과하는 최소 구현이 작성됨
- [ ] GREEN: 모든 테스트가 통과함
- [ ] REFACTOR: 코드 품질이 개선됨
- [ ] REFACTOR: 리팩토링 후에도 테스트가 통과함

### 수용 기준
- [ ] 모든 acceptance_criteria가 검증됨
- [ ] 충족되지 않은 기준이 명확히 보고됨
- [ ] 완료된 티켓만 completed 상태로 변경됨

### 산출물
- [ ] output_artifacts에 명시된 모든 파일이 생성됨
- [ ] 테스트 파일이 test_file_path에 생성됨
- [ ] 생성된 파일이 프로젝트 구조에 맞게 배치됨

---

## 5. 에러 처리 검증

### 실패 처리
- [ ] 테스트 실패 시 재시도 로직이 작동함
- [ ] 최대 재시도 횟수가 준수됨
- [ ] 실패한 티켓이 failed 상태로 표시됨
- [ ] 실패 원인이 명확히 기록됨

### 복구 옵션
- [ ] 실패한 티켓 재실행 옵션이 제공됨
- [ ] 수동 수정 옵션이 제공됨
- [ ] 스킵 옵션이 제공됨

---

## 6. 프로그레스 추적 검증

### 상태 업데이트
- [ ] 티켓 상태가 실시간으로 업데이트됨
- [ ] 프로그레스 파일이 각 단계마다 저장됨

- [ ] 실패한 티켓에 대한 액션 플랜이 있음
- [ ] 최종 리포트가 생성됨
