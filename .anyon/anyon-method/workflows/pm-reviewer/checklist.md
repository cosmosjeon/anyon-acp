# PM Reviewer 검증 체크리스트

## 1. 사전 조건 검증

### 필수 입력 파일
- [ ] execution-progress.md 파일이 존재함
- [ ] 완료된 Epic이 최소 1개 이상 존재함
- [ ] Epic 관련 커밋이 존재함

### 설계 문서 로드
- [ ] Architecture 문서가 로드됨
- [ ] TRD 문서가 로드됨
- [ ] API Spec 문서가 로드됨 (있는 경우)

---

## 2. 리뷰 대상 파악 검증

### 변경 파일 분류
- [ ] Backend 파일이 식별됨
- [ ] Frontend 파일이 식별됨
- [ ] Test 파일이 식별됨
- [ ] Config 파일이 식별됨

### 리뷰 범위
- [ ] 모든 변경 파일이 리뷰 대상에 포함됨
- [ ] 파일 수가 정확히 집계됨

---

## 3. 병렬 리뷰 실행 검증

### 4개 리뷰어 실행
- [ ] 코드 품질 리뷰어가 실행됨
- [ ] 아키텍처 리뷰어가 실행됨
- [ ] 보안 리뷰어 (OWASP)가 실행됨
- [ ] 테스트 커버리지 리뷰어가 실행됨

### 병렬 실행
- [ ] 4개 리뷰어가 동시에 실행됨 (순차 아님)
- [ ] 모든 리뷰어 결과가 수집됨

---

## 4. 코드 품질 리뷰 검증

### 체크 항목
- [ ] 중복 코드 (DRY) 검사됨
- [ ] 네이밍 컨벤션 검사됨
- [ ] 함수/클래스 크기 검사됨
- [ ] 코드 복잡도 검사됨
- [ ] 불필요한 코드 검사됨

### 결과 형식
- [ ] 이슈가 YAML 형식으로 출력됨
- [ ] 각 이슈에 file, line, type, description 포함됨
- [ ] can_auto_fix 여부가 명시됨

---

## 5. 아키텍처 리뷰 검증

### 체크 항목
- [ ] 폴더 구조 준수 여부 검사됨
- [ ] 레이어 분리 검사됨
- [ ] 의존성 방향 검사됨
- [ ] 설계 패턴 준수 검사됨

### 기준 적용
- [ ] Architecture 문서의 규칙이 적용됨
- [ ] 위반 사항에 expected 값이 명시됨

---

## 6. 보안 리뷰 검증 (OWASP)

### 체크 항목
- [ ] SQL Injection 검사됨
- [ ] XSS 검사됨
- [ ] 입력 검증 검사됨
- [ ] 인증/인가 검사됨
- [ ] 민감정보 노출 검사됨
- [ ] 에러 노출 검사됨

### 심각도 분류
- [ ] Critical 이슈가 식별됨 (있는 경우)
- [ ] High 이슈가 식별됨 (있는 경우)
- [ ] Medium 이슈가 식별됨 (있는 경우)

---

## 7. 테스트 커버리지 리뷰 검증

### 체크 항목
- [ ] 테스트 파일 존재 여부 검사됨
- [ ] 정상 케이스 테스트 존재 검사됨
- [ ] 에러 케이스 테스트 존재 검사됨
- [ ] 엣지 케이스 테스트 존재 검사됨

### 누락 테스트 식별
- [ ] 테스트 없는 구현 파일이 식별됨
- [ ] 추가해야 할 테스트가 제안됨

---

## 8. 이슈 통합 및 분류 검증

### 이슈 통합
- [ ] 4개 리뷰 결과가 통합됨
- [ ] 전체 이슈 수가 집계됨

### 분류
- [ ] auto_fixable 이슈가 분류됨
- [ ] manual_required 이슈가 분류됨
- [ ] security_critical 이슈가 분류됨 (있는 경우)

---

## 9. 즉석 수정 검증

### 자동 수정
- [ ] auto_fixable 이슈에 대해 수정 시도됨
- [ ] 수정 후 테스트 실행됨
- [ ] 테스트 통과 시 fixed_issues에 추가됨

### 실패 처리
- [ ] 3회 실패 시 failed_issues에 추가됨
- [ ] 실패한 파일이 롤백됨
- [ ] 실패 원인이 기록됨

### WebSearch 활용
- [ ] 수정 방법 모를 때 WebSearch 사용됨
- [ ] 보안 이슈는 WebSearch로 모범 사례 검색됨

---

## 10. 커밋 검증

### 수정사항 커밋
- [ ] 수정된 이슈가 있으면 커밋 생성됨
- [ ] 커밋 메시지에 수정 내용 포함됨
- [ ] 커밋 메시지에 "Generated by PM Reviewer" 포함됨

---

## 11. 결과 출력 검증

### 필수 출력
- [ ] Epic ID와 리뷰 완료 메시지 출력됨
- [ ] 수정된 이슈 목록 출력됨
- [ ] 수동 확인 필요 이슈 목록 출력됨
- [ ] 문제없는 영역 목록 출력됨

### Progress 파일 업데이트
- [ ] 리뷰 결과가 progress 파일에 기록됨
- [ ] 리뷰 일시가 기록됨
- [ ] 이슈 수가 기록됨

---

## 12. 최종 검증

### 완성도
- [ ] 모든 변경 파일이 리뷰됨
- [ ] 자동 수정 가능한 이슈가 모두 처리됨
- [ ] 수동 확인 필요 이슈가 명확히 보고됨

### 다음 단계 안내
- [ ] 다음 Epic 실행 안내가 출력됨
- [ ] 또는 프로젝트 완료 시 최종 리포트 안내됨
