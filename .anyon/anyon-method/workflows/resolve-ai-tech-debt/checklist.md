# AI 코드 기술부채 분석 - 검증 체크리스트

이 체크리스트는 기술부채 분석 보고서의 품질과 완전성을 검증합니다.

---

## 1. 프로젝트 스캔 및 베이스라인

### 범위 설정
- [ ] 분석 대상 디렉토리가 명확히 지정됨
- [ ] 제외 패턴(node_modules, vendor 등)이 적용됨
- [ ] 주요 프로그래밍 언어가 식별됨

### 베이스라인 측정
- [ ] 총 파일 수 및 라인 수가 측정됨
- [ ] 언어별 분포가 파악됨
- [ ] 디렉토리 구조가 문서화됨
- [ ] 기존 린트 경고/에러 수가 기록됨

---

## 2. 죽은 코드 분석

### 미사용 Import
- [ ] 모든 import/require 문이 분석됨
- [ ] 미사용 import가 파일명, 라인번호와 함께 나열됨
- [ ] 순환 import가 검사됨

### 미사용 함수/메서드
- [ ] 정의되었지만 호출되지 않는 함수가 식별됨
- [ ] export 되었지만 외부에서 사용되지 않는 함수 검사됨
- [ ] 각 함수의 영향도(low/medium/high)가 평가됨

### 미사용 변수/상수
- [ ] 선언 후 사용되지 않는 변수가 식별됨
- [ ] 할당 후 읽히지 않는 변수가 검사됨
- [ ] 미사용 매개변수가 나열됨

### 미사용 클래스/타입
- [ ] 인스턴스화되지 않는 클래스가 식별됨
- [ ] 사용되지 않는 타입 정의가 나열됨

---

## 3. 코드 품질 분석

### 복잡도 분석
- [ ] 순환 복잡도가 측정됨 (권장: 10 이하)
- [ ] 함수당 라인 수가 검사됨 (권장: 20줄 이하)
- [ ] 중첩 깊이가 검사됨 (권장: 3레벨 이하)
- [ ] 복잡도 높은 함수가 구체적으로 나열됨

### 코드 중복
- [ ] 5줄 이상의 중복 코드 블록이 식별됨
- [ ] 중복률(%)이 계산됨
- [ ] 추상화 가능한 반복 로직이 제안됨

### 코드 스멜
- [ ] Long Method가 검사됨
- [ ] Large Class가 검사됨
- [ ] Feature Envy 패턴이 검사됨
- [ ] Data Clumps가 식별됨
- [ ] 각 코드 스멜에 대한 수정 권장사항 포함

### 명명 규칙
- [ ] 일관된 명명 규칙 사용 여부 검사됨
- [ ] 의미 없는 변수명(x, temp, data 등)이 식별됨
- [ ] 언어별 컨벤션 준수 여부 검사됨

---

## 4. 테스트 커버리지 분석

### 테스트 현황
- [ ] 테스트 디렉토리 구조가 파악됨
- [ ] 테스트 프레임워크가 식별됨
- [ ] 테스트 파일 수가 기록됨

### 커버리지 측정
- [ ] 라인 커버리지가 측정됨 (또는 측정 불가 사유 명시)
- [ ] 테스트가 없는 핵심 함수가 식별됨
- [ ] 복잡도 높은데 테스트 없는 코드가 나열됨

### 테스트 우선순위
- [ ] 테스트 추가 우선순위가 제안됨
- [ ] 비즈니스 로직 핵심 함수가 강조됨

---

## 5. 아키텍처 분석

### 의존성 분석
- [ ] 모듈 간 의존성 구조가 파악됨
- [ ] 순환 의존성이 검사됨
- [ ] 불필요한 의존성이 식별됨

### 결합도/응집도
- [ ] 높은 결합도를 가진 모듈이 식별됨
- [ ] 단일 책임 원칙 위반 사례가 나열됨

### 패턴 일관성
- [ ] 사용 중인 디자인 패턴이 식별됨
- [ ] 안티패턴 사용 사례가 검사됨
- [ ] 패턴 일관성 이슈가 문서화됨

---

## 6. 레거시 코드 분석

### Deprecated API
- [ ] deprecated 기능 사용이 식별됨
- [ ] 대체 API가 제안됨
- [ ] 구버전 라이브러리 의존성이 나열됨

### 구식 패턴
- [ ] 현대적 대안이 있는 구식 패턴이 식별됨
- [ ] 마이그레이션 방안이 제시됨

### 보안 취약점
- [ ] 알려진 취약점 있는 의존성이 검사됨
- [ ] 하드코딩된 비밀정보가 검사됨
- [ ] 안전하지 않은 함수 사용이 검사됨

### 성능 이슈
- [ ] N+1 쿼리 패턴이 검사됨
- [ ] 비효율적인 알고리즘이 식별됨

---

## 7. 데이터베이스 기술부채 분석

### 스키마 분석
- [ ] 미사용 테이블이 식별됨
- [ ] 미사용 컬럼이 식별됨
- [ ] 인덱스 최적화 필요 영역이 분석됨
- [ ] 정규화 이슈가 검토됨
- [ ] FK 제약조건 누락/불일치가 검사됨

### 쿼리 분석
- [ ] N+1 쿼리 패턴이 검사됨
- [ ] 비효율적 쿼리(LIKE '%...', SELECT *, 누락된 인덱스 등)가 식별됨
- [ ] Raw SQL과 ORM 혼용 패턴이 검토됨
- [ ] 쿼리 성능 영향도가 평가됨

### 마이그레이션 분석
- [ ] 미적용 마이그레이션이 확인됨
- [ ] 스키마 정의와 실제 DB 스키마 불일치가 검사됨
- [ ] 롤백 가능성이 평가됨
- [ ] 마이그레이션 히스토리 일관성이 확인됨

### ORM/모델 분석
- [ ] 미사용 모델이 식별됨
- [ ] 관계 정의(FK, 역참조 등) 이슈가 검토됨
- [ ] 모델 유효성 검증 규칙이 검사됨
- [ ] Lazy/Eager Loading 전략이 분석됨
- [ ] N+1 Loading 문제가 확인됨

### 데이터 무결성 분석
- [ ] 고아 레코드(FK 참조 없는 레코드)가 식별됨
- [ ] 중복 데이터 패턴이 검사됨
- [ ] NULL 처리 일관성이 검토됨
- [ ] 데이터 타입 일관성이 확인됨
